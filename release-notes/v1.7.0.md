# Kaspa Sovereign Architect Engine v1.7.0 - Complete Toccata Engineering Update

This repository version updates the full skill package for Rusty Kaspa
`v2.0.0` and the scheduled Toccata mainnet activation at DAA `474,165,565`.
It does not claim that `v1.7.0` has been published or that Toccata is active
before the live mainnet DAA reaches the threshold.

## Skill Contract

- Rebuilt `SKILL.md` as a concise senior-engineering operating contract.
- Enforced UTXO-first and DAG-aware architecture.
- Enforced explicit fees, recipients, change, custody, signing intent, and key
  boundaries.
- Preserved Kasware and Kaspium compatibility.
- Required `kaspa:` and `kaspatest:` address validation.
- Separated protocol activation from wallet, indexer, pool/miner, SDK,
  explorer, and application readiness.

## Toccata Intelligence

- Tracks final release `v2.0.0`, activation DAA, live mainnet DAA, UTC estimate,
  P2P protocol version cutoff, and network identity.
- Classifies upstream branch deltas by consensus, transaction format,
  covenant, RPC/WASM, wallet/PSKT, mempool, storage/IBD, security, and test
  impact.
- Documents `storageMass`/`storage_mass`, legacy JSON `mass`, and
  `compute_commit` compatibility.
- Documents the relay-policy fee formula, one-way database migration, and
  pool/miner block-template field preservation.

## Cross-Platform Adapters

- Aligned Codex, OpenAI, Claude, Cursor, OpenClaw, Gemini CLI, and generic
  adapters to the same safety and activation contract.
- Removed the stale Cursor `v1.4` label.
- Made adapter exports self-contained with `SKILL.md`, references, and assets.

## Validation And Packaging

- Added deterministic source-intelligence tests to release packages.
- Expanded compatibility validation to detect missing Toccata, wallet, fee,
  address-prefix, UTXO, DAG, and verification rules in any adapter.
- Added extracted-package smoke tests to compatibility and release workflows.
- Release artifacts now prove that source monitoring, knowledge drills,
  covenant lineage, vProg scope, network checks, and readiness gates run from
  the package itself.
